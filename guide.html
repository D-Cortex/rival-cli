rival CLI

Push function code, manage versions, and interact with the Rival platform — all from your terminal.

Node ≥ 18 TypeScript Descope Auth

Installation

# npm
npm install -g rival-cli
# pnpm
pnpm add -g rival-cli
rival --version   # confirm install

Quick Start

# 1. Authenticate
rival login

# 2. Create a new function (scaffolds files + saves to backend)
rival init

# 3. Edit your code, then push
rival push

# 4. Fetch latest code from the backend
rival load

# 5. List all functions in your org
rival fetch

Config Files

Two files control the CLI's behaviour:

FilePurpose
~/.rival/config.json Global auth — token, refresh token, org ID, email, API URL. Written by rival login.
~/.rival/.env Optional env overrides loaded at every CLI startup before the project .env.
rival.json Per-project function config — function ID, slugs, selected version, runtime, files. Lives in your project folder.

Authentication

rival login Authenticate via email OTP (Descope)

Sends a one-time code to your Rival email, verifies it with Descope, then fetches your organizations and saves credentials to ~/.rival/config.json.

rival login
rival login --email you@company.com
FlagDescriptionDefault
-e, --email <email>Skip the email promptinteractive
-u, --api-url <url>Override the backend API URLcortexone-api-dev
ℹ

If you belong to multiple organizations you will be prompted to select one. You can change org later with rival login again or by editing ~/.rival/config.json.

rival logout Remove stored credentials

Deletes ~/.rival/config.json, clearing your token, refresh token, and org ID.

rival logout
rival whoami Verify session and show identity

Calls GET /api/v1/users/me to validate your token live (not just check if a token is stored). Displays your email, name, org, and a list of all your organizations.

rival whoami

✓ Authenticated

  Email:   you@company.com
  Name:    Aryan Bhandari
  Org ID:  org_abc123
  API:     https://cortexone-api-dev.rival.io

  Organizations:
    ●  Acme Corp (acme)  ← active
    ○  Personal (personal)
FlagDescription
-u, --api-url <url>Override the backend API URL
ℹ

If the token has expired but a refresh token is stored, the CLI automatically obtains a new session token and retries. You only see an error if the refresh also fails.

Functions

rival init Create a new function on Rival and scaffold local files

Interactively collects function metadata, calls POST /api/v1/functions, writes backend-returned starter files to disk, and creates rival.json.

rival init
rival init --cwd ./my-function

Prompts collected during init:

  • Function name
  • Short description
  • Runtime (Python 3.13 / JavaScript / Lua)
  • Tool type (Forecast, Indicator, …)
  • Category
  • Sector(s) — multi-select
  • Compute type (CPU / GPU)
FlagDescription
--cwd <dir>Run init inside a specific directory (default: process.cwd())
-u, --api-url <url>Override the backend API URL
⚠

rival init will exit if rival.json already exists in the target directory. Delete it first if you want to re-initialise.

rival push Upload local files to Rival

Reads rival.json, validates each file against the runtime's allowed extensions, then calls PUT /api/v1/functions/{id}/save-version.

# Push using the version stored in rival.json
rival push

# Push to an explicit version
rival push --version v2

# Push from a specific directory
rival push --cwd ./my-function
FlagDescriptionDefault
--version <ver>Version label to save torival.json → "Draft"
--cwd <dir>Working directoryprocess.cwd()
-u, --api-url <url>Override the backend API URLcortexone-api-dev
ℹ

Use rival select first to pin a version in rival.json — then every rival push will target that version automatically.

rival load Pull latest code from Rival and write files to disk

Fetches the function version stored in rival.json (or the first available version) from GET /api/v1/function/{orgSlug}/{fnSlug}/details and writes each file to the working directory. Also updates the files list in rival.json.

rival load

✓  main.py          (text/x-python, 42 lines)
✓  requirements.txt (text/plain, 5 lines)

2 file(s) written to /Users/you/my-function
FlagDescription
--cwd <dir>Write files to this directory (default: process.cwd())
-u, --api-url <url>Override the backend API URL
⚠

Existing files with the same name will be overwritten. Commit or stash local changes before running rival load.

rival fetch List all functions in your organization

Calls GET /api/v1/functions/summary and prints a table of every function in your active org.

rival fetch

Each row shows: name · slug · ID · type · visibility · runtime · latest version · state · description.

FlagDescription
-u, --api-url <url>Override the backend API URL
rival select Pin a version in rival.json

Fetches all versions for the current function and prompts you to choose one. Saves the selection to rival.json so subsequent rival push and rival load calls target that version automatically.

rival select

? Select a version  (Use arrow keys)
❯ Draft
  v1
  v2
FlagDescription
--cwd <dir>Working directory containing rival.json
-u, --api-url <url>Override the backend API URL

rival.json schema

Created by rival init, updated by rival select and rival load. Must be in the working directory when running rival push, rival load, or rival select.

// rival.json { "functionId": "fn_abc123", // backend function ID "orgSlug": "acme", // org slug from backend "fnSlug": "my-price-forecast", // function slug "version": "Draft", // active version label "runtime": "python:3.13", // runtime identifier "files": [ // relative file paths "main.py", "requirements.txt" ] }

Environment variables

Loaded from ~/.rival/.env first, then .env in the package directory. Values here override defaults but are overridden by CLI flags.

VariableDescriptionDefault
RIVAL_TOKEN Session JWT — skips reading from config.json —
RIVAL_ORG_ID Organization ID override —
RIVAL_API_URL Backend API base URL cortexone-api-dev.rival.io
DESCOPE_PROJECT_ID Descope project ID used for OTP auth P34BHLas1imEyKtqssBSuha6Hbxu
